Field FAQ

Quick answers to the questions that come up every time

The FAQ addresses the most common field questions in plain language — internet, apps, power, device compatibility, privacy, and what happens when things go wrong.

When someone is confused in the field, give them the next action — not a lecture. Each answer below is short by design. If a longer explanation is needed, it belongs in a full guide, not a FAQ. Point to the handout, the cable, the folder, or the next step.

Common questions and plain answers

Does it need internet to work?

No. The local library runs completely offline. Once the Raspberry Pi is powered on and a phone connects to its Wi-Fi, all content is available without any internet connection. The core library works in fully offline settings. Satellite, when used, is a separate receive-and-replay path for Free-to-Air broadcasts — not required for normal library access.

Does every phone need a special app?

No. Basic library access through the Raspberry Pi Wi-Fi just needs a browser — nothing extra to install. Some sharing methods (like LocalSend) use a free app for cross-device transfers, but for joining the VillageServer library, a standard browser is all that's needed.

What happens if power fails mid-session?

The session pauses — but files that were already downloaded to phones are still accessible offline. This is why the setup guide emphasizes downloading files to the phone during the session, not just streaming from the server. If the server goes off, the downloaded content stays.

Can local leaders run this without tech skills?

Yes — that's the design goal. The setup process is six steps. The printed guides use plain language, not technical terms. The system is intentionally simple enough for someone who has never managed a computer before. If a local leader struggles to follow the guide, the guide needs to improve — not the person.

What if the Wi-Fi name doesn't appear?

Give the Pi 90 seconds to fully boot before checking. If the network still doesn't appear: confirm the Pi is receiving power (check the LED indicator), try turning the phone's Wi-Fi off and back on, and check that the power source is stable. Most "not appearing" problems are power-related.

What if a file won't open?

Try a different file first to confirm the connection is working. If one file fails but others open, the specific file may be corrupted — delete and re-download it. If no files open, the problem is the connection or browser, not the content. Clear the browser cache and try again.

How do we update the content?

Content is updated before deployment or anywhere normal internet is available. In the field, a personal hotspot or local Wi-Fi can be used when practical. Satellite broadcasts are handled separately as received or recorded media, not as the normal Pi update path.

How to handle confusion in the field

  1. Listen for the real blocker

    Is the problem power, Wi-Fi connection, file location, device type, language, or confidence? Identify the actual issue before suggesting a fix.

  2. Give the shortest accurate answer first

    One sentence. If the answer takes three sentences to explain, the person probably needs to see it done, not described.

  3. Show the next action on the actual device

    Point to the button, cable, Wi-Fi name, folder, or handout. Have them do the step themselves while you watch — not while you do it for them.

  4. Confirm on the device before moving on

    The answer doesn't count until the person has done the step and it worked. "I understand" is not the same as "it opened."

  5. If multiple people ask the same question, fix the guide

    Repeated questions mean the current explanation isn't working. Update the PDF or printed handout so the answer is there the next time.
