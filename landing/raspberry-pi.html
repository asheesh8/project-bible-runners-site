Technology · Raspberry Pi VillageServer

A library that fits in your hand — full setup guide

The Raspberry Pi broadcasts a local Wi-Fi hotspot and serves offline Bible libraries to any nearby phone. This guide walks through flashing the device, loading language libraries, and optionally setting up a WireGuard tunnel so a technician can manage the server remotely from anywhere in the world.

What the VillageServer interface looks like

When a phone joins the VillageServer Wi-Fi and opens 10.43.0.1 in a browser, it sees a clean library home page. The interface is fully offline — no internet required.

The home page shows the available language libraries. In Eric's demo unit, both English and Spanish libraries are loaded. Each language tile opens into a content grid with four sections:

  • Bible — full Scripture text, chapter by chapter
  • Audio — dramatised audio Bible recordings
  • Video — LUMO gospel films (Matthew, Mark, Luke, John) and other gospel media
  • Books — commentaries, discipleship materials, and reference PDFs

When content is being updated remotely, the interface displays an "Updating Libraries — please wait" banner so users know a refresh is in progress. The server continues to serve existing content while the update runs in the background.

How language libraries are loaded

Each microSD card carries one language library. The card sits inside a USB reader or adapter, and those readers plug into the Raspberry Pi's USB ports. Swapping or adding a card changes the available languages immediately on the next server restart.

This means a kit can be prepared before travel with the right languages already on the cards. In the field, the operator only needs to join Wi-Fi and open a browser — no understanding of servers is required.

Part 1 — Setting up the Raspberry Pi server

What you need

  • Raspberry Pi 4 (2 GB RAM or more recommended) or Raspberry Pi 3B+
  • MicroSD card for the Pi OS (16 GB minimum, 32 GB recommended)
  • One or more USB microSD readers loaded with language library cards
  • Power supply (5V USB-C for Pi 4, or a compatible power bank)
  • A laptop with Raspberry Pi Imager installed (free download from raspberrypi.com)
  1. Step 1 Flash Raspberry Pi OS to the system card

    Download and open Raspberry Pi Imager on your laptop. Choose Raspberry Pi OS Lite (64-bit) — the version without a desktop, which runs lighter and faster as a server. Select your system microSD card as the destination.

    Before writing, click the settings gear and configure:

    • Hostname: villageserver
    • Enable SSH with password authentication
    • Username: pi, Password: choose a strong password and note it
    • Wi-Fi credentials for your setup laptop's network (so the Pi can reach the internet on first boot)

    Write the image and insert the card into the Raspberry Pi.

  2. Step 2 First boot and SSH in

    Power on the Pi. Wait about 60 seconds for first boot. Find the Pi's IP address from your router's device list (look for hostname villageserver), then connect:

    # Replace 192.168.x.x with your Pi's actual IP ssh pi@192.168.x.x

    Accept the fingerprint prompt and enter the password you set in Imager. You are now at the Pi's command line.

  3. Step 3 Update the system
    sudo apt update && sudo apt upgrade -y

    This takes a few minutes on first run. Let it complete before moving on.

  4. Step 4 Install Kiwix and the VillageServer web interface

    Kiwix is the offline library engine that serves the content libraries from ZIM files stored on the microSD cards.

    # Install Kiwix server sudo apt install -y kiwix-tools # Install nginx as a front-end reverse proxy sudo apt install -y nginx # Create the content mount point sudo mkdir -p /mnt/library
  5. Step 5 Mount the language library cards

    Plug the USB readers (with language cards inserted) into the Pi's USB ports. List the connected devices to find them:

    lsblk -o NAME,SIZE,FSTYPE,MOUNTPOINT,LABEL

    Each card will appear as a device like sda, sdb, etc. Mount them so Kiwix can read the ZIM files:

    # Example: mount first card to /mnt/library/english sudo mkdir -p /mnt/library/english sudo mount /dev/sda1 /mnt/library/english # Repeat for each language card sudo mkdir -p /mnt/library/spanish sudo mount /dev/sdb1 /mnt/library/spanish

    To make mounts persist across reboots, add them to /etc/fstab using the card's UUID (find it with blkid).

  6. Step 6 Start the Kiwix server

    Point Kiwix at the ZIM files on the mounted cards and start it as a background service:

    # Start Kiwix on port 8080, scanning all ZIM files under /mnt/library kiwix-serve --library --port=8080 /mnt/library/**/*.zim &

    Create a systemd service file to start Kiwix automatically on boot:

    sudo nano /etc/systemd/system/kiwix.service

    Paste the following, then save:

    [Unit] Description=Kiwix VillageServer After=network.target [Service] ExecStart=/usr/bin/kiwix-serve --library --port=8080 /mnt/library/**/*.zim Restart=on-failure User=pi [Install] WantedBy=multi-user.target
    sudo systemctl daemon-reload sudo systemctl enable kiwix sudo systemctl start kiwix
  7. Step 7 Configure nginx to serve the library on port 80

    Edit the default nginx site to proxy requests to Kiwix:

    sudo nano /etc/nginx/sites-available/default

    Replace the contents with:

    server { listen 80 default_server; server_name _; location / { proxy_pass http://127.0.0.1:8080; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; } }
    sudo nginx -t && sudo systemctl reload nginx
  8. Step 8 Set up the Wi-Fi hotspot

    Install the hostapd access point daemon and dnsmasq for DHCP:

    sudo apt install -y hostapd dnsmasq sudo systemctl unmask hostapd

    Configure the hotspot. Edit /etc/hostapd/hostapd.conf:

    sudo nano /etc/hostapd/hostapd.conf
    interface=wlan0 driver=nl80211 ssid=VillageServer hw_mode=g channel=7 wmm_enabled=0 macaddr_acl=0 auth_algs=1 ignore_broadcast_ssid=0 wpa=2 wpa_passphrase=village123 wpa_key_mgmt=WPA-PSK wpa_pairwise=TKIP rsn_pairwise=CCMP

    Configure dnsmasq to give connected phones the address 10.43.0.x and point DNS to the Pi:

    sudo nano /etc/dnsmasq.conf
    interface=wlan0 dhcp-range=10.43.0.2,10.43.0.50,255.255.255.0,24h address=/#/10.43.0.1

    Give wlan0 a static IP and enable IP forwarding:

    sudo nano /etc/dhcpcd.conf

    Add at the end:

    interface wlan0 static ip_address=10.43.0.1/24 nohook wpa_supplicant
    sudo systemctl enable hostapd dnsmasq sudo systemctl restart dhcpcd sudo systemctl start hostapd dnsmasq
  9. Step 9 Test the library from a phone

    On a phone, go to Wi-Fi settings and connect to VillageServer with password village123. Open a browser and go to 10.43.0.1. You should see the language library grid. Tap a language, then a content section, and verify that Bible text, audio, or video loads without any internet connection.

Part 2 — Remote access via WireGuard tunnel

Once deployed in a village, the Pi may be unreachable from the outside world — behind a cellular NAT, a private router, or simply a dynamic IP that changes daily. WireGuard solves this by having the Pi connect outward to a relay server (VPS) that you control. You also connect to that same VPS. Both the Pi and your laptop are on the same private WireGuard network, so you can SSH into the Pi as if it were sitting next to you.

What you need for this section
  • A VPS (virtual private server) with a public IP — any provider works (DigitalOcean, Linode, Vultr, AWS, etc.). The cheapest tier ($4–6/month) is sufficient.
  • Ubuntu 22.04 or Debian 12 on the VPS
  • WireGuard installed on: the VPS, the Pi, and your laptop

How the tunnel works

Each device gets a private WireGuard IP on a shared subnet (e.g., 10.99.0.0/24):

  • VPS (relay): 10.99.0.1
  • Raspberry Pi (field device): 10.99.0.2
  • Your laptop (admin): 10.99.0.3

The Pi and your laptop both connect to the VPS at boot. To reach the Pi you SSH to 10.99.0.2 — the VPS relays the packets transparently.

  1. VPS · Step 1 Install WireGuard on the VPS
    sudo apt update && sudo apt install -y wireguard sudo mkdir -p /etc/wireguard cd /etc/wireguard # Generate VPS keypair wg genkey | tee vps_private.key | wg pubkey > vps_public.key chmod 600 vps_private.key
  2. VPS · Step 2 Create the VPS WireGuard config
    sudo nano /etc/wireguard/wg0.conf
    [Interface] Address = 10.99.0.1/24 ListenPort = 51820 PrivateKey = <contents of vps_private.key> PostUp = iptables -A FORWARD -i wg0 -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE PostDown = iptables -D FORWARD -i wg0 -j ACCEPT; iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE # Raspberry Pi peer [Peer] PublicKey = <Pi public key — generated in next step> AllowedIPs = 10.99.0.2/32 # Admin laptop peer [Peer] PublicKey = <Laptop public key — generated later> AllowedIPs = 10.99.0.3/32
    # Enable IP forwarding on the VPS echo "net.ipv4.ip_forward=1" | sudo tee -a /etc/sysctl.conf sudo sysctl -p # Start WireGuard on the VPS sudo systemctl enable wg-quick@wg0 sudo systemctl start wg-quick@wg0

    Make sure port 51820/udp is open in the VPS firewall (UFW: sudo ufw allow 51820/udp).

  3. Pi · Step 1 Install WireGuard on the Pi
    sudo apt install -y wireguard sudo mkdir -p /etc/wireguard cd /etc/wireguard # Generate Pi keypair wg genkey | tee pi_private.key | wg pubkey > pi_public.key chmod 600 pi_private.key # Print the public key — copy it into the VPS config above cat pi_public.key
  4. Pi · Step 2 Create the Pi WireGuard config
    sudo nano /etc/wireguard/wg0.conf
    [Interface] Address = 10.99.0.2/24 PrivateKey = <contents of pi_private.key> DNS = 1.1.1.1 [Peer] PublicKey = <contents of VPS vps_public.key> Endpoint = <VPS_PUBLIC_IP>:51820 AllowedIPs = 10.99.0.0/24 PersistentKeepalive = 25

    PersistentKeepalive = 25 sends a keepalive packet every 25 seconds so the tunnel stays open even through cellular NAT.

    # Enable the tunnel — starts at boot, connects out to VPS sudo systemctl enable wg-quick@wg0 sudo systemctl start wg-quick@wg0 # Verify the tunnel is up sudo wg show

    You should see latest handshake for the VPS peer if everything is configured correctly. Go back to the VPS and add the Pi's public key to its wg0.conf if you haven't already, then run sudo wg addconf wg0 <(wg-quick strip wg0) or restart the VPS tunnel.

  5. Laptop · Step 1 Install WireGuard on your admin laptop

    On macOS, download the WireGuard app from the Mac App Store. On Linux:

    sudo apt install -y wireguard

    Generate a keypair and create your client config:

    wg genkey | tee laptop_private.key | wg pubkey > laptop_public.key
    [Interface] Address = 10.99.0.3/24 PrivateKey = <contents of laptop_private.key> DNS = 1.1.1.1 [Peer] PublicKey = <contents of VPS vps_public.key> Endpoint = <VPS_PUBLIC_IP>:51820 AllowedIPs = 10.99.0.0/24 PersistentKeepalive = 25

    Add your laptop's public key to the VPS wg0.conf under a new [Peer] block, then reload the VPS tunnel:

    # On the VPS after editing wg0.conf sudo systemctl restart wg-quick@wg0
  6. Laptop · Step 2 Connect and SSH into the field Pi

    Activate the WireGuard tunnel on your laptop (toggle it on in the WireGuard app, or run sudo wg-quick up wg0 on Linux). Then SSH directly to the Pi's WireGuard IP:

    ssh pi@10.99.0.2

    You are now on the Pi's command line from anywhere in the world — no port-forwarding, no dynamic DNS, no public IP on the Pi required.

Part 3 — Updating content libraries remotely

Once you have SSH access through the WireGuard tunnel, you can push new or updated ZIM files to the Pi without physically touching it.

  1. Step 1 Copy a new ZIM file to the Pi over the tunnel
    # Run this on your laptop, not the Pi scp new-library-english.zim pi@10.99.0.2:/home/pi/incoming/
  2. Step 2 Move the file and restart Kiwix
    # SSH into the Pi first (see Part 2, Step 2) ssh pi@10.99.0.2 # Move the new ZIM to the library mount sudo mv /home/pi/incoming/new-library-english.zim /mnt/library/english/ # Restart Kiwix to pick up the new file sudo systemctl restart kiwix

    The VillageServer interface shows an "Updating Libraries" banner to connected phones while Kiwix restarts. Within about 30 seconds the new content is live.

  3. Step 3 Verify from a connected phone

    On a phone already connected to the VillageServer Wi-Fi, reload 10.43.0.1 and confirm the new or updated library tile appears. Tap through to verify the content opens correctly.

Quick troubleshooting

  • Phone connects to VillageServer Wi-Fi but can't reach 10.43.0.1 — Check that nginx and Kiwix are running: sudo systemctl status nginx kiwix. Also confirm the wlan0 static IP is set correctly with ip addr show wlan0.
  • WireGuard tunnel shows no handshake — Confirm the VPS has port 51820/udp open and the public keys are copied correctly into each peer's config. Run sudo wg show on both the Pi and VPS to compare.
  • Kiwix returns no libraries — The ZIM files may not be mounted. Run df -h to confirm the USB cards are mounted, and ls /mnt/library/ to confirm ZIM files are present.
  • Pi loses Wi-Fi hotspot after update — This can happen if wlan0 is reconfigured by dhcpcd during a package update. Recheck /etc/dhcpcd.conf and restart hostapd: sudo systemctl restart hostapd.
