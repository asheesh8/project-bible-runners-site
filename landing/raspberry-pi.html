Local server

The Raspberry Pi makes the library local.

The Pi is the small computer inside VillageServer. It stores the offline library, creates the local Wi‑Fi network, and serves a simple website that nearby phones can open without mobile data.

Simple modelPartsField rulesSet it upGuides

Simple model

People do not connect to the internet. They connect to the kit.

That distinction matters. A phone may say “No internet,” but the local library can still be working correctly.

Power starts the server

Solar, battery, or local power starts the Raspberry Pi and the library service.

Wi‑Fi appears nearby

The kit broadcasts a local VillageServer network for phones and tablets.

Browser opens the library

Users open a local address or captive prompt, then choose a language and resource.

Files can leave the kit

People save useful files and share them by phone, USB, or microSD.

What each part does

Small hardware, clear responsibilities.

Pi

Server brain

Runs the local website and organizes content so users see a simple library instead of folders of confusing files.

SD

System card

Holds the operating setup. It should stay with the server and should not be swapped casually in the field.

USB

Storage and updates

Can carry media libraries, language packs, or field updates depending on the kit configuration.

Field rules

Make startup boring and repeatable.

  • VentilationKeep the Pi dry, shaded, and able to breathe.
  • Approved powerUse the labeled power source so the server does not brown out.
  • Wait before testingGive the server time to start before deciding Wi‑Fi failed.
  • One test phoneAlways verify with one saved file before inviting a group.

Set it up

From a cold kit to a working library in five steps.

No servers to think about, no internet account to create. Power it on, join its Wi‑Fi, and open the library in any browser. Tap through the steps below.

  1. 1Power on
  2. 2Find Wi‑Fi
  3. 3Open library
  4. 4Browse & save
  5. 5Share on

1 · Power the kit on

Connect the labeled power source — solar, battery, or local mains. A small light shows the Raspberry Pi is starting. Give it about 60 seconds to boot the library before testing.

Need the power details? See Power systems & solar solutions →

2 · Find the local Wi‑Fi

On any phone or tablet, open Settings → Wi‑Fi and look for the VillageServer network. Join it like any other Wi‑Fi. Your phone may say “No internet” — that is normal. You are connecting to the kit, not the internet.

3 · Open the library

Open any web browser. Most phones show the library automatically; if not, type the local address:

http://villageserver.local

4 · Browse and save

Choose a language, then read Bibles, listen to audio Scripture, watch teaching videos, and open training — all served from the kit. Tap save / download to keep a file on the phone so it works even after walking away.

5 · Share it onward

Saved files can move phone‑to‑phone with no kit nearby. That is how one library reaches a whole village.

See Phone‑based Gospel distribution → and the step‑by‑step transfer guides →
