Board brief - optional uplink

Satellite Systems and Content Acquisition

Satellite can help a team update, download, or communicate from remote places, but the local library should still work without it.

Plain answer

Satellite is optional. It is helpful for updates and remote coordination, but it should not become the foundation. VillageServer is strongest when the local library works offline first.

Field steps

Use it this way

  1. Decide if it is neededUse satellite only when updates, coordination, or downloads cannot wait for normal internet.
  2. Choose the locationFind open sky, stable ground, and a safe cable path.
  3. Align and powerSet up the dish, connect power, and wait for the link to stabilize.
  4. Download intentionallyUpdate only the resources needed for the mission, not random large files.
  5. Return to offline modeMake sure the local library still works after the satellite link is turned off.
  6. Document the sessionWrite down what was updated and any connection problems.

Best use

Remote content updates, coordination, emergency communication, or proof-of-concept work.

Not required

The core library should still work without satellite.

Main risks

Power draw, sky view, weather, subscription cost, and setup complexity.

Board takeaway

Satellite is a bridge for updates, not the heart of the system.
